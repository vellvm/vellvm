# Interpreter performance stress tests

Each file in this directory isolates one cost center of the executable
semantics, so that before/after timings localize the impact of a change.
They all carry a deterministic `; ASSERT EQ:` so they also work as plain
correctness tests, but their point is wall-clock time:

```sh
cd src && make perf       # build ./vellvm if needed, then time each test
```

(equivalently, run `./perf/run.sh` from the repo root), or a single one:

```sh
cd src && /usr/bin/time -p ./vellvm -test-file ../perf/loop-phi-arith.ll
```

Interpreter startup (parse + elaboration + global init) is ~0.2 s; every
test is calibrated to run for a few seconds so that startup noise is
negligible. To make a test heavier or lighter, change the loop bound
passed in `@main` (each file's header comment gives the closed form for
the expected result; update the `ASSERT EQ` accordingly).

| test | stresses | scales with |
|---|---|---|
| `loop-phi-arith.ll` | block denotation, `br`, phi resolution, local read/write; no memory events. The baseline: every other test pays this too. | loop bound |
| `wide-arith.ll` | i128 arithmetic (add/mul/shl/icmp) with values kept above 64 bits: the per-width integer machinery (modulus/natwordsize recomputation, multi-limb Z normalization). Compare per-iteration cost against `loop-phi-arith.ll`. | loop bound |
| `locals-chain.ll` | local-env growth: straight-line code, one fresh binding per instruction, every read searches the env. Quadratic under an alist env, ~n·log n under a tree. Generated by `gen-locals-chain.py`. | number of temporaries (regenerate) |
| `calls-fib.ll` | Call/Return events, argument binding, frame push/pop (naive doubly-recursive fib ⇒ exponentially many calls). | fib argument |
| `block-jumps.ll` | per-jump block-target resolution: a tight loop placed after 6400 unreachable blocks, so every jump resolves its target against the whole function. ~9× slower under a linear `find_block` scan than with the AVL block map; the dead blocks never execute, so the delta vs. `loop-phi-arith.ll` is pure lookup cost. Generated by `gen-block-jumps.py`. | dead-block count / loop bound (regenerate) |
| `calls-large-fn.ll` | per-call denotation setup vs. callee size: a trivial callee padded to 400 unreachable blocks, called 40000 times. Catches per-call work proportional to the callee's block count — e.g. building the block map per call instead of once per definition in `denote_function` (~4× here, far worse at larger sizes). Generated by `gen-calls-large-fn.py`. | callee block count / call count (regenerate) |
| `calls-many-fns.ll` | per-call function-pointer dispatch vs. module size: 5000 trivial functions, the last-defined one called 40000 times. `lookup_defn` resolves the callee in an AVL IntMap built once in `TopLevel`, so per-call cost is flat in the number of definitions (measured ~65 → ~70 µs/call from N=50 to N=5000); an association-list dispatch or a per-call map rebuild makes this blow up. Generated by `gen-calls-many-fns.py`. | function count / call count (regenerate) |
| `switch-cases.ll` | switch dispatch vs. case count: a loop over a 256-case `switch`. `denote_terminator` re-elaborates every case literal per execution before the linear scan. The elaboration now runs in EOU under a single lift (was a per-case itree `map_monad`, ~0.6 µs/case of pure bind machinery); the remaining ~1.4 µs/case is `repr`'s bignum `mod 2^sz` per literal (`P_mod_two_p`) — eliminating it needs per-function pre-elaboration or cheaper integer ops (cf. `wide-arith.ll`). Generated by `gen-switch-cases.py`. | case count / loop bound (regenerate) |
| `mem-scan.ll` | scalar load/store throughput: byte serialization/deserialization + GEP into a flat i64 array. | loop bound (= array size) |
| `mem-aggregate.ll` | serialization of structured values: store/load of a mixed `{i64, i32, [4 x i64], i8}` struct plus insertvalue/extractvalue chains. | loop bound |
| `memcpy-chunk.ll` | bulk memory copy — the byte-level *read* path: 100 memcpys of 8 KiB. Guards the linearity of bulk reads: `read_bytes` used to collect bytes with the value-collecting `map_monad`, making the copy quadratic in its size (this size ran 72 s); now an accumulator fold (`map_monad_acc`), linear. The rest is the per-byte AVL traffic — `memset-chunk.ll` is the write-only floor to compare against. | buffer size / loop bound |
| `memset-chunk.ll` | bulk memory write — the byte-level *write* path: 100 memsets of 8 KiB. Linear (`loop_monad`) but pays the one-boxed-MByte-per-byte representation, ~2–3 µs/byte: the floor for all bulk memory traffic. | buffer size / loop bound |
| `alloca-churn.ll` | allocation: one `alloca` per iteration ⇒ fresh-address generation, provenance, frame-stack growth; detects degradation as live allocations accumulate. | loop bound |
| `alloca-large.ll` | large-object allocation + frame teardown: 200 calls each alloca'ing 4 KiB. Allocation writes one undef byte per address and registers every byte's pointer in the frame; the return frees them one AVL delete at a time (~13 ms/call). | buffer size / call count |
| `iptr-roundtrip.ll` | `ptrtoint`/`inttoptr` round trips and loads through recovered pointers: the ITOP/PTOI + provenance machinery. | loop bound |
| `undef-pick.ll` | the uvalue side: values kept symbolic (`undef` + select), stored (symbolic byte serialization) and branched on (Pick/concretization). | loop bound |
| `vector-ops.ll` | vector element access: a loop-carried `<64 x i64>` with one insertelement + one extractelement per iteration. Vectors are dvalue lists, so lane access is a linear walk and each insert rebuilds the vector. (`shufflevector` is unimplemented and untestable.) | loop bound / vector width |
| `global-init.ll` | startup initialization of a `[16384 x i64] zeroinitializer` global: denoting the aggregate + serializing 128 KiB through the byte-level write path. All startup, ~68 µs/element. CAUTION: at `[32768 x i64]` initialization stack-overflows (non-tail-recursive aggregate path) — this file doubles as a canary for that threshold. | array size |

Reference timings (Apple Silicon, July 2026, after switching the
local/global envs to AVL maps, fixing the quadratic pending-bind
accumulation in `map_monad_`/`denote_code`, switching `denote_ocfg`'s
block lookups to a per-definition AVL block map, fixing the
quadratic bulk-read path with `map_monad_acc` in `read_bytes`/
`get_consecutive_ptrs`, and deferring source-location string formatting
to read time — [set_loc] now stores the raw [file_info], worth ~2% on
instruction-dense tests):
loop-phi-arith 5.3 s · calls-fib 5.8 s · mem-scan 4.3 s · mem-aggregate
2.3 s · alloca-churn 1.8 s · iptr-roundtrip 3.1 s · undef-pick 2.8 s ·
wide-arith 4.3 s · locals-chain 0.24 s · block-jumps 5.7 s ·
calls-large-fn 2.7 s · calls-many-fns 3.7 s · switch-cases 3.5 s (was
4.7 s before moving case elaboration into one lifted EOU computation) ·
memcpy-chunk 3.7 s (at 8 KiB;
was 72 s before the map_monad_acc fix) · memset-chunk 2.3 s ·
alloca-large 2.6 s · vector-ops 2.1 s ·
global-init 1.1 s. 

These are for orientation only — always re-measure the baseline on your own machine
before comparing.

This directory deliberately lives outside `tests/` so that `make test`
(which recursively picks up every `.ll` under `tests/`) does not run it;
use `make perf` instead.
